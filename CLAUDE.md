# Shift Rota — working notes for Claude

## Repo shape
- The app is one file: everything lives in `index.html` (inline CSS + JS, Supabase JS client).
- PWA files sit alongside it: `manifest.webmanifest`, `sw.js` (service worker), and
  `icon-192.png` / `icon-512.png` / `apple-touch-icon.png`. The app is installable to the
  home screen. Bump the `CACHE` version in `sw.js` if you change what the shell caches.
- `db-setup.sql` holds the Supabase RLS policies / schema the app depends on.
- Served by GitHub Pages from `main`. No build step.
- The `.ics` calendar export is generated by a **separate Cloudflare Worker** (not in this repo).

## Workflow — ALWAYS sync main first
Before starting any change, sync `main` into the working branch:

```
git fetch origin main
git merge origin/main   # or rebase
```

Why: `index.html` is sometimes edited directly on github.com (commits titled
"Update index.html …"). Those land on `main` only, so a feature branch can fall
behind and conflict. Syncing first avoids surprise merge conflicts at PR time.

## Conventions
- Develop on the designated feature branch, commit, push, open a PR, then merge to `main`.
- After editing the inline `<script>`, sanity-check JS syntax with `node --check` on the extracted script.
- Two importers already exist on the Templates page (admin only): "Import duty sheets
  from PDF" (per-duty) and "Import a full rota" (multi-week). Don't remove either.
